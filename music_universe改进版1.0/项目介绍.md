# Music Universe（音乐宇宙）改进版 — 项目介绍

## 1. 项目定位

本仓库面向「数据处理与可视化」课程实践，将 **Spotify 曲目音频特征**、**Billboard 冠军单曲及其歌词主题** 等数据打通：一方面用 Python 完成清洗、统计与静态/交互图表产出；另一方面用浏览器中的 **Three.js 3D 场景** 把 Spotify 与 Billboard 样本放在同一套可切换的「宇宙」视图里探索，并支持筛选、模式切换与预览相关能力。

改进版相对原型的主要增强包括：

- **独立 Web 应用**：`web/music_universe/` 下可本地或静态托管打开的交互页，数据以内嵌 JS 或 JSON 形式供给前端。
- **元数据增强脚本**：`code/enrich_music_metadata.py` 通过 **iTunes Search API**（必要时回退 **Deezer**）为曲目补充公开 **封面图** 与 **30 秒试听 URL**，结果写回 `music_universe_data_embedded.js`，并缓存到 `output/tables/`。
- **脚本加载容错**：`index.html` 中 Three.js / OrbitControls 优先加载本地 `vendor/`，失败时回退 CDN；脚本加载失败会在页面 `errorBanner` 中提示。
- **分析管线日志**：`music_hit_analysis.py` 将运行过程写入 `output/logs/music_hit_analysis_v3.log`，便于复现与排错。

## 2. 目录结构（摘要）

| 路径 | 说明 |
|------|------|
| `data/` | 原始 CSV：`spotify_tracks.csv`、`billboard.csv`、`topics.csv` |
| `code/music_hit_analysis.py` | 主分析脚本：清洗、汇总表、Matplotlib/Seaborn 图、Plotly 交互 HTML、摘要报告 |
| `code/enrich_music_metadata.py` | 读取/写回 `window.SITE_DATA = {...}` 形式的数据文件，联网查询并更新 `preview_url`、`image_url` 等 |
| `code/export_site_data_json.py` | 将内嵌 JS 中的 JSON 导出为 `music_universe_data_embedded.json`，便于校验或配合 `app.js` 的 fetch 回退 |
| `requirements.txt` | 分析脚本 Python 依赖清单 |
| `output/cleaned_data/` | 清洗后的中间表 |
| `output/tables/` | 汇总 CSV、iTunes 缓存 `itunes_metadata_cache.json` 等 |
| `output/figures/` | 静态图（`spotify/`、`billboard/`、`comparison/`）与 Plotly 导出的 `interactive_html/` |
| `output/reports/analysis_summary_report.md` | 流水线自动生成的文字摘要 |
| `output/logs/` | 分析脚本运行日志 |
| `web/music_universe/` | 前端：`index.html`、`app.js`、`styles.css`、`vendor/`（Three r128）、`music_universe_data_embedded.js`（体积大，为内嵌 JSON）、`serve_web.py`（本地 HTTP） |

## 3. 数据分析管线（`music_hit_analysis.py`）

### 3.1 功能概览

- **多编码读取 CSV**：依次尝试 `utf-8-sig`、`utf-8`、`gbk`、`gb18030`、`cp932`、`big5`、`latin1`，降低中文 Windows / 导出工具导致的乱码问题。
- **清洗与衍生**：Spotify 侧保留 popularity、danceability、energy、valence、tempo、genre 等；Billboard 侧整合年代、周数、能量/可舞度等专家评分字段，并与 `topics.csv` 做主题展开。
- **可视化**：箱线图、热力图、小提琴图、网络/桑基类交互图等，输出到 `output/figures/` 各子目录。
- **报告**：`output/reports/analysis_summary_report.md` 中强调 **关联不等于因果**，适合直接引用到课程论文的方法论段落。

### 3.2 路径与环境变量

默认数据路径为仓库内 `data/*.csv`；换路径时可用环境变量覆盖：

- `MUSIC_SPOTIFY_PATH` — Spotify CSV
- `MUSIC_BILLBOARD_PATH` — Billboard CSV
- `MUSIC_TOPICS_PATH` — 主题列表 CSV
- `MUSIC_OUTPUT_ROOT` — 输出根目录（默认：项目根下的 `output`）
- `MUSIC_SPOTIFY_MAX_ROWS` — 可选，正整数时对 Spotify 清洗结果随机下采样，便于低配机冒烟测试

依赖见根目录 `requirements.txt`。安装示例：

```bash
pip install -r requirements.txt
```

### 3.3 运行方式

在已配置 Python 依赖的环境中执行（默认已指向本仓库 `data/`）：

```bash
python code/music_hit_analysis.py
```

完成后检查 `output/logs/` 与 `output/reports/` 是否生成/更新。

## 4. 交互前端（`web/music_universe/`）

### 4.1 技术栈

- **Three.js（r128）** + **OrbitControls**：3D 星点/球体、雾效、自动旋转与阻尼轨道控制。
- **纯静态资源**：无构建步骤；推荐用 **`python web/music_universe/serve_web.py`** 起本地服务后再用浏览器打开（`file://` 下部分功能受限，见《问题排查》）。页面在 `file://` 时会显示可关闭的琥珀色提示条，而非红色错误条。

### 4.2 数据加载逻辑（`app.js`）

- 若全局已有 `window.SITE_DATA`（由先加载的 `music_universe_data_embedded.js` 赋值），则直接使用。
- 否则尝试 `fetch("./music_universe_data_embedded.json")`；仓库默认以 **内嵌 JS** 为主路径，JSON 为可选备用。

`SITE_DATA` 中预期包含例如：`spotifyItems`、`billboardItems`、`genres`、`stats`、`billboardStats` 等字段，每条曲目可带 `modes`（`universe`、`galaxyCluster`、`coverWall`、`genreRings`、`billboardDecades` 等坐标）。

### 4.3 交互模式（界面「Universe Controls」）

- **Universe**：以 danceability、energy、valence 等构成主坐标系。
- **Galaxy Cluster**：在浏览器内对特征做类 PCA 的协方差迭代投影（若数据里已预计算 `modes.galaxyCluster` 则优先使用）。
- **Cover Wall / Genre Rings / Billboard Decades**：海报墙、流派环、Billboard 按年代等布局切换。

筛选支持：流派、关键词、人气或周数阈值、数据集切换（Spotify / Billboard）、演示轮播等。

### 4.4 音频与预览

- **Play Signature**：无公开 `preview_url` 时，用 **Web Audio** 根据 tempo、energy、valence、danceability 生成短促「声纹」示意（非原曲）。
- **Play Preview**：若元数据中含 iTunes/Deezer 提供的试听 URL，则通过 `HTMLAudioElement` 播放（受跨域与浏览器自动播放策略影响，见《问题排查》）。

## 5. 元数据增强（`enrich_music_metadata.py`）

- 读取 `web/music_universe/music_universe_data_embedded.js`，要求文件以固定前缀 `window.SITE_DATA = ` 开头，内容为合法 JSON。
- 对每条曲目构造查询词，请求 iTunes；若预览或封面不完整，可再查 Deezer。
- 结果合并进内存中的 `SITE_DATA`，写回同一 JS 文件；同时在 `output/tables/itunes_metadata_cache.json`（及 CSV）中缓存，避免重复请求。

常用参数：`--limit`（限制本次新查询条数）、`--country`（iTunes 店面）、`--sleep`（请求间隔）、`--workers`（并发数）、`--retry-failed`（对缓存中无预览的条目重试）。

## 6. 与课程作业的衔接建议

- **数据处理**：突出清洗规则、缺失与异常处理、分层（如 popularity 分档）与对照表构建。
- **可视化**：静态图用于论文插图；Plotly HTML 与 Music Universe 页用于答辩演示。
- **伦理与边界**：报告与界面文案均已提示勿将统计关联写成因果；试听仅使用平台允许的公开预览链。
